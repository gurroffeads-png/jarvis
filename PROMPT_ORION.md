## 1. O que e o Orion
Orion e um **assistente pessoal de IA em portugues**, sendo transformado num **produto SaaS vendavel**.
Nasceu de um assistente pessoal (J.A.R.V.I.S) e foi repaginado: marca propria (sem IP de terceiros),
voz brasileira, interface estilo site (abas/paginas internas), planos mensais e checkout.

## 2. Arquitetura (3 pecas independentes)
1. **App desktop** - `jarvis_servidor.py` (backend Python) + `orion_app.html` (interface SPA).
   Empacotado como `dist/Orion/Orion.exe` (PyInstaller, spec em `jarvis.spec`).
   Cerebro: **Ollama local** (gratis, padrao) ou chave propria (Anthropic). Tem voz (edge-tts),
   microfone, abrir apps do PC, etc. Dados do usuario em `%APPDATA%\Orion` (sobrevivem a atualizacao).
2. **App de nuvem** - `orion_cloud.py` (servidor multi-inquilino, so stdlib do Python + SQLite).
   Cada usuario isolado por **sessao em cookie assinado**. Cerebro na nuvem **plugavel** (API
   compativel com OpenAI: Groq gratis, OpenAI, etc. via env `LLM_API_KEY`/`LLM_BASE_URL`/`LLM_MODEL`).
   Serve a MESMA `orion_app.html` em "modo nuvem" (injeta `window.ORION_CLOUD=true`).
   Deploy via Docker (`Dockerfile` + `render.yaml`) no Render/Railway/Fly.
   PARIDADE (testado local): a nuvem ja tem Aprender, Trading IA (cotacao Yahoo ao vivo + paper trading + analise + analise de print por VISAO Groq via `cloud_vision`), tudo por usuario. Auto-agente e Binance real ficam so no desktop (por seguranca). Pasta `nuvem/` tem todos os arquivos prontos pro GitHub.
3. **Site de vendas** - `site/` (estatico, gerado por `_build_site.py` a partir de `orion_site.html`).
   Vai no **Netlify** (no ar: orionassistenteia.netlify.app). Tem lista de espera (Netlify Forms) e
   pagina `obrigado.html`. Liga ao app pela const `APP_URL` no `site/index.html`.

## 3.1. Modulo Trading IA (novo)
No `jarvis_servidor.py` (rotas `/trading`, `/trading/analise`, `/trading/cotacao`) + pagina "Trading IA" no `orion_app.html`.
- **Cotacoes ao vivo** via Yahoo Finance (chart v8, sem auth): B3 = `TICKER.SA`, cripto = `BTC-USD`. Cache de 20s.
- **Grafico TradingView** embutido (widget gratis `s3.tradingview.com/tv.js`, tema escuro, simbolo `BMFBOVESPA:PETR4` / `BINANCE:BTCUSDT`).
- **Analise por IA** (botao): o cerebro le preco + medias moveis e devolve leitura + vies COMPRA/VENDA/NEUTRO (com disclaimer: nao e recomendacao).
- **Paper trading** (simulacao, R$100k virtual): comprar/vender, posicoes com P&L ao vivo, historico, KPIs (patrimonio/lucro/saldo).
- **Agente automatico** (simulacao): regra de cruzamento de medias (SMA 9 vs 21) opera a watchlist sozinho num loop de 1 min; o usuario so assiste. Dados em `jarvis_trading.json`.
- **2 modalidades** na pagina: (a) "Por ativo (ao vivo)" = grafico + analise + paper trading; (b) "Por print" = usuario envia/cola o SCREENSHOT de um grafico e a IA de VISAO devolve plano estruturado (ATIVO/TENDENCIA/SUPORTE/RESISTENCIA/ENTRADA/STOP/ALVO/FUNDAMENTO). Endpoint `/trading/analise_imagem`, funcao `trading_analise_imagem` + `_visao_responder`.
- LIMITACAO REAL: a leitura de print depende do modelo de visao. llava/moondream LOCAL le mal grafico (resposta generica).
- VISAO PLUGAVEL (`_visao_responder`): prioridade Anthropic (se cerebro=anthropic+chave) > **Groq** (chave gratis do usuario, cred `groq_key`, modelo `groq_vision_model` default llama-3.2-90b-vision-preview, endpoint OpenAI-compativel) > Ollama llava/moondream local. Campo "Visao / analise de imagem" em Configuracoes pra colar a chave Groq. Com Groq a leitura de grafico fica boa, de graca. Status `groq` em integra.status().
- **OPERACOES REAIS (Binance)**: card "Operacoes reais" no modo ativo. Conecta a conta Binance do usuario (binance_key/binance_secret nas integracoes), mostra saldo, e permite ordens MARKET de cripto. SEGURANCA (responsavel, semi-automatico): toda ordem passa primeiro pelo endpoint de TESTE da Binance (`/api/v3/order/test`, valida sem executar), mostra os detalhes, e so executa de verdade (`/api/v3/order`) depois que o usuario CONFIRMA. Teto de seguranca `REAL_TETO_USD=200` por ordem. Funcoes `_binance_signed`, `binance_preco` (publico), `binance_conta`, `binance_ordem`. Rotas `/binance/conta`, `/binance/ordem`. NAO ha robo autonomo disparando ordem real sozinho (risco + responsabilidade do usuario + CVM se vender pra terceiros). Acoes B3 reais nao dao porque corretoras de varejo nao tem API aberta; por isso real e so cripto via Binance.
- IMPORTANTE: e tudo SIMULACAO. Execucao real com dinheiro de verdade NAO esta implementada (regulacao CVM, API de corretora, risco). Caminho futuro: semi-automatico (usuario confirma cada ordem) na corretora do proprio usuario.

## 3.2. Modulo Aprender (tutor de IA) - novo
Pagina "Aprender" + rotas `/aprender`, `/aprender/trilha`, `/aprender/licao`. O usuario diz um objetivo
(ex: "criar um site sem ser programador", "investir do zero", "criar um app com vibe coding"), a IA monta
uma TRILHA de passos e ENSINA cada passo (licao com o que e / por que importa / como fazer + tarefa pratica),
em linguagem de leigo. Progresso por usuario em `jarvis_aprender.json`. Pilar pra "ajudar pessoas a aprender",
foco em nao-desenvolvedores / vibe coders.

- Gera chave no `OrionKeys.exe` OU no app (dono > Conta > Admin > "Gerar chave").

## 6. Pagamento
- **Mercado Pago** (Pix/boleto/cartao) e **PayPal**. Checkout criado no servidor.
- Desktop: token em Configuracoes > Pagamento (so o dono ve). Nuvem: env `MP_ACCESS_TOKEN` / `PAYPAL_*`.
- ATENCAO: Mercado Pago recusa `auto_return` quando a back_url e `localhost` (da HTTP 400).
  So usar `auto_return` com https (nuvem). No desktop, sem auto_return.

## 7. Como rodar / buildar / deployar
- **Rodar desktop (dev):** `py -3.12 jarvis_servidor.py` (ou `_serve_only.py` so pra servir a UI em http://localhost:8765).
- **Buildar o exe:** `py -3.12 -m PyInstaller jarvis.spec --noconfirm --clean` -> `dist/Orion/Orion.exe`.
- **Gerador de chaves:** `py -3.12 -m PyInstaller orion_keygen.py --onefile --windowed --name OrionKeys --hidden-import orion_licenca --icon orion.ico`.
- **Rodar nuvem (dev):** `set LLM_API_KEY=gsk_...` e `py -3.12 orion_cloud.py` (porta 8766).
- **Deploy nuvem:** Groq (chave gratis) > GitHub (sobe a pasta `nuvem/`) > Render (Blueprint le `render.yaml`, cola `LLM_API_KEY`).
- **Site:** arrasta `site/` no Netlify. Liga ao app editando `APP_URL` no `site/index.html`.
- **APK:** com o app no ar, `pwabuilder.com` gera o APK/AAB (PWA ja pronto: manifest, sw, icone, /.well-known/assetlinks.json).
